Geoprocessing

Geoprocessing functions operate on the geometry of features

Functions fall into 3 categories:

  1. geometric measurements (area and length)
  2. creating new geometries
  3. tests for spatial proximity or relationships (covered under spatial queries)


Geometric Measurements

  • st_area() Area
  • st_length() Length / perimeter
  • st_distance() Distance between features

Results are returned in map units!

Unit info is saved, so you can convert to other units using units::set_units()


New geometries

Functions that take a single spatial input:

  • st_buffer()
  • st_centroid()
  • st_convex_hull()
  • st_voronoi()
  • st_simplify()

Two sf inputs:

  • st_union()
  • st_intersection()
  • st_difference()


Lists

  • list objects in R are very flexible
  • elements can be of different classes (even other lists)
  • elements can be named
one_shoe <- list(type = "New Balance", size = 9, condition = "good")

one_shoe$type
## [1] "New Balance"
  • you can also grab elements by index number
one_shoe[[3]]
## [1] "good"


Common list operations:

  • add an element to a list (assign a value to the new element as if it existed and it will be created on-the-fly)
  • delete an element from a list (assign it NULL)
  • join lists (use c())

Spatial Queries

Tests for spatial proximity or relationships

  • st_intersects(x, y) Overlaps or touches
  • st_touches(x, y) Touches only
  • st_within(x, y) Within
  • st_contains(x, y) Contains
  • st_overlaps(x, y) Overlaps
  • st_is_within_distance(x, y) Within a distance


These functions return i) logical values, or ii) row numbers depending on the value of sparse

sparse = TRUE: results are packaged in a list of indices (one element for every feature in x)

sparse = FALSE: get back a matrix of logical values (m x n)


Using spatial relationship tests for queries

List results - use slice():

test_results_lst <- x_sf %>% st_touches(y_sf, sparse = TRUE)

y_features_touching_ith_feature_in_x <- y_sf %>% slice(test_results_lst[[i]])


Logical matrix - use filter():

test_results_mat <- x_sf %>% st_touches(y_sf, sparse = FALSE)

y_features_touching_ith_feature_in_x <- y_sf %>% filter(test_results_mat[,i])


Spatial Queries using [square bracket notation]

xy_intersection <- x_sf[y_sf, ]
